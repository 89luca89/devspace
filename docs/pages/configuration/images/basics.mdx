DevSpace can build, tag and push images using a variety of build tools.

:::tip <u>Don't</u> Build Images During Development
Because image building can waste quite a bit of time, we recomment that you only build images when running DevSpace in your CI/CD pipelines using `devspace build/deploy` but not when a developer runs `devspace dev`.

Instead of image building, the use the following workflow for `devspace dev`:
1. Deploy your application using prod/stable images
2. Use the [`devImage` feature](../#TODO) to swap out the prod/stable images in runtime using a prebuilt dev-optimized image.
3. Use the [`sync` feature](../#TODO) for hot reloading your containers
:::


## Workflow
To build, tag and push images with DevSpace, you need to:
1. Define your images in the `images` section of `devspace.yaml`
2. Run the `build_images` function inside the `pipelines` section of `devspace.yaml`


### 1. Define `images`
Define as many images as need in your `devspace.yaml`:
```yaml
images:
  backend:
    dockerfile: ./Dockerfile
  frontend:
    dockerfile: ./frontend/Dockerfile
    context: ./frontend
  other: ...
```

- via `devspace run-pipeline build`
- via `devspace build` (shortcut)
